# Steps to Create a Custom Manifest Validator

1. **Define the Manifest Schema**
   - List required top-level properties (modules, initialState, schemaVersion)
   - Specify types for each property
   - Detail structure for nested objects/arrays (e.g., modules array)

2. **Write Validation Functions**
   - Check if manifest is an object
   - Check for presence of required properties
   - Validate types of each property
   - For arrays, check each item for required fields and types

3. **Error Reporting**
   - Collect and format validation errors
   - Provide clear error messages for missing/invalid fields

4. **Testing**
   - Create test cases for valid and invalid manifests
   - Ensure all schema rules are enforced

5. **Integration**
   - Replace AJV usage in DataLoader.js with the custom validator
   - Ensure error handling and reporting is consistent

6. **Documentation**
   - Document the schema and validation logic for future maintenance
